# 医院内外部人员智能管理系统
基于计算机视觉技术的医院人员综合管理解决方案，整合**人脸识别考勤**与**行人流量监控**两大核心能力，实现医院内部员工身份验证、自动打卡签到，以及公共区域人流量实时统计与可视化分析，有效提升医院人员管理效率与安全保障水平。
#### 视频链接：https://www.bilibili.com/video/BV1CgLg6gE3J/?spm_id_from=333.1387.upload.video_card.click&vd_source=7d115318c28581cfeb0e7d3a4a21aaf4
## ✨ 核心功能
### 1. 人像录入系统
- 支持录入员工**姓名、职位、部门**完整信息
- 自动检测人脸并裁剪保存，自动分配唯一编号
- 录入完成后自动训练LBPH人脸识别模型，生成`train.yml`模型文件
- 实时预览摄像头画面，支持中文信息叠加显示

### 2. 身份验证系统
- 实时摄像头人脸识别，快速匹配已录入人员
- 自动区分**内部员工**与**未知访客(unknown)**
- 人脸区域圆形标注+姓名显示，直观清晰
- 可自定义识别置信度阈值，平衡准确率与召回率

### 3. 员工打卡系统
- 人脸识别自动签到，无需人工操作
- 自动记录签到时间，防止重复签到
- 每日生成独立签到记录文件（`signed_in_YYYYMMDD.txt`）
- 实时显示当前时间与签到状态（`姓名 OK 签到时间`）

### 4. 行人检测与人流量统计
- 基于YOLOv5s轻量模型，实时检测视频流中的行人
- 实时显示当前画面内行人数量
- 自动保存每帧人流量数据到`people_counts.txt`
- 生成人流量随时间变化的折线图，直观展示客流规律

## 📦 环境配置
### 1. 安装依赖
```bash
# 基础依赖
pip install opencv-python opencv-contrib-python pysimplegui pillow numpy matplotlib

# PyTorch CPU版本（有GPU请替换为对应CUDA版本）
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
```
### 2. 必要文件准备
- 项目根目录需放置haarcascade_frontalface_alt2.xml人脸检测分类器（可从 OpenCV 官方仓库下载）
- 首次运行行人检测时，程序会自动下载 YOLOv5s 预训练模型（约 14MB）
### 3.创建必要目录
```bash
mkdir -p face/2 face/3
```
- face/2/：存储每日签到记录文件
- face/3/：存储录入的人脸图像数据

## 📁 项目目录结构
```plaintext
hospital-personnel-management/

│   └── 3/          # 人脸图像目录

```
## 🚀 使用说明
### 1. 启动系统
```bash
python 整体代码.py
```
将弹出图形化主界面，包含四个功能按钮。
### 2. 人像录入（首次使用必须执行）
- 点击【录入人像系统】
- 输入员工姓名，选择职位和部门，点击【确定】
- 调整摄像头角度，确保人脸清晰出现在画面中
- 按s键保存人脸图像（建议录入 5-10 张不同角度、不同光线的照片）
- 按q键退出录入，程序将自动训练人脸识别模型
### 3. 员工打卡
- 点击【员工打卡】
- 员工面对摄像头，系统将自动识别并完成签到
- 首次签到会显示姓名 OK 签到时间，重复签到仅显示姓名与当前时间
- 按q键退出打卡界面
### 4. 身份验证
- 点击【身份验证系统】
- 系统将实时识别摄像头前的人员，显示姓名或unknown
- 按q键退出验证界面
### 5. 行人检测与人流量分析
- 点击【行人检测并查看行人流量图】
- 系统将打开摄像头实时检测行人，左上角显示当前人数
- 按ESC键停止检测，程序将自动生成并显示人流量变化曲线图
## ⚠️ 注意事项
- **摄像头权限**：确保程序拥有摄像头访问权限，笔记本电脑需开启内置摄像头
- **字体问题**：中文显示依赖系统的STKAITI.TTF（华文楷体）和SimHei（黑体）字体，Linux/Mac 系统需手动安装对应字体
- **识别阈值**：人脸识别置信度阈值默认设为 80-85，可根据实际环境调整（值越高越严格）
- **模型训练**：新增人员后必须重新运行录入系统完成模型训练，否则无法识别
- **文件路径**：所有文件需保持在同一目录下，请勿随意修改文件夹名称
- **性能优化**：行人检测对 CPU 要求较高，建议使用性能较好的设备运行
